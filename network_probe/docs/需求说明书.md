# 网络拨测工具 NetProbe — 需求说明书

**文档版本：** v1.1  
**编写日期：** 2026-03-24  
**项目名称：** 小D网络拨测工具 NetProbe  
**目标平台：** Windows（免安装单机运行）

---

## 1. 项目概述

### 1.1 项目背景

在网络运维和故障排查过程中，运维人员经常需要对目标网络节点进行连通性测试、DNS 解析验证、HTTP 服务可用性检测以及 TCP 连接稳定性评估。现有的命令行工具（如 ping、nslookup、curl）虽然功能齐全，但操作分散、缺乏统计分析能力，且不适合长时间持续监控。

本项目旨在开发一款**集成化的 Windows 桌面 GUI 网络拨测客户端工具**，将 Ping、DNS、HTTP(S)、TCP 长连接四种拨测协议整合在统一界面中，提供即时拨测和长时间拨测两大核心功能，并具备实时数据展示、统计分析和日志导出能力。

### 1.2 项目目标

- 提供简洁易用的图形化界面，降低网络拨测操作门槛
- 支持 **Ping / DNS / HTTP(S) / TCP 长连接** 四种主流拨测协议
- 提供即时拨测（手动启停）和长时间拨测（定时自动完成）两种工作模式
- 即时拨测和长时间拨测均支持全部四种协议
- 实时显示拨测结果，支持历史记录回看
- 长时间拨测结束后自动生成统计分析报告和质量评级
- 支持拨测日志一键导出
- 最终交付为 **Windows 免安装 exe 程序**，可直接双击运行

### 1.3 面向用户

- 网络运维工程师
- 系统管理员
- 开发测试人员
- 需要排查网络问题的普通用户

### 2.2 即时拨测功能

#### 2.2.1 基本描述

即时拨测提供快速的网络探测能力，用户输入目标地址和端口并选择协议后，即可启动持续拨测，手动停止后结束。

#### 2.2.2 输入参数

| 参数 | 说明 | 约束 |
|------|------|------|
| 目标地址 | `地址:端口` 格式，支持 IPv4/IPv6/域名 | 必填，格式校验，端口范围 1~65535 |
| 拨测方式 | Ping / DNS / Curl / 长连接 四选一 | 默认选中 Ping |

**输入格式示例：**
- `8.8.8.8:53`（IPv4 + 端口）
- `www.baidu.com:443`（域名 + 端口）
- `[::1]:80`（IPv6 + 端口）

#### 2.2.3 拨测协议详情

**Ping 协议**
- 使用系统 `ping` 命令实现 ICMP 探测
- 采集指标：RTT（往返时延，ms）、TTL、状态（成功/超时/不可达/错误）
- 默认参数：间隔 200ms，超时 2000ms，包大小 64 字节，TTL 64
- 支持 Windows 和 Linux 双平台命令适配
- 注：Ping 使用 ICMP 协议，端口参数不生效但保留以统一输入格式

**DNS 协议**
- 使用 dnspython 库进行 DNS 查询
- 采集指标：查询耗时（ms）、解析结果 IP、RCODE、TTL
- 支持自定义 DNS 服务器
- 支持多种查询类型（A、AAAA、CNAME、MX 等）
- 默认参数：间隔 500ms，超时 2000ms
- 限制：仅支持域名输入，不支持 IP 地址
- 注：DNS 查询固定使用端口 53，用户输入的端口不影响 DNS 行为

**Curl/HTTP(S) 协议**
- 使用 requests 库进行 HTTP/HTTPS 请求
- 采集指标：DNS 耗时、TCP 连接耗时、TLS 握手耗时、首字节时间（TTFB）、传输耗时、总耗时、HTTP 状态码
- 自动根据端口判断协议：端口 443 使用 HTTPS，其他使用 HTTP
- 支持 GET 请求方法
- 支持跟随重定向和 TLS 证书校验控制
- 默认参数：间隔 1000ms，连接超时 5000ms，总超时 10000ms

**TCP 长连接协议**
- 纯 TCP 保活探测模式，不发送应用层数据
- 工作流程：建立 TCP 连接 → 开启 OS 级 TCP keepalive → 周期性检测连接存活 → 连接断开后自动重连
- 采集指标：建连 RTT、心跳探活状态、会话时长、断线事件、重连等待时间
- 连接存活检测机制：使用 `select()` + 非阻塞 `recv()` 探测 socket 状态
- 断线重连策略：指数退避（基础 1 秒，最大 30 秒）+ ±20% 随机抖动
- 默认参数：心跳间隔 2000ms，心跳超时 3000ms，建连超时 5000ms
- 适用场景：评估 TCP 连接稳定性、检测服务端空闲超时策略

#### 2.2.4 实时结果展示

- 拨测结果以日志形式实时追加显示在暗色终端风格文本框中
- 成功记录显示为绿色（#a6e3a1），失败/超时记录显示为红色（#f38ba8）
- 支持自动滚动到最新结果；用户手动上翻时暂停自动滚动
- 提供"回到最新"按钮快速定位到最新记录
- 每条记录格式包含：时间戳、序号、各协议特定指标、状态
- TCP 长连接额外显示：事件类型图标（🔗建连/💓心跳/🔌断线/⏳重连）、会话 ID、会话时长

#### 2.2.5 状态栏

实时显示：已探测次数、成功次数、丢包次数、近 10 次平均 RTT、运行时长。

### 2.3 长时间拨测功能

#### 2.3.1 基本描述

长时间拨测适用于需要持续监控一段时间网络质量的场景。用户设定拨测时长后启动，到时间后自动停止并输出统计报告。支持全部四种拨测协议。

#### 2.3.2 输入参数

| 参数 | 说明 | 约束 |
|------|------|------|
| 目标地址 | `地址:端口` 格式 | 必填，格式校验 |
| 拨测时长 | 持续拨测时间 | 1~1440 分钟（最长 24 小时），默认 5 分钟 |
| 拨测方式 | Ping / DNS / Curl / 长连接 四选一 | 默认选中 Ping |

#### 2.3.3 进度展示

- 显示进度条，实时更新完成百分比
- 显示剩余时间
- 进度条颜色根据实时丢包率动态变化：
  - 蓝色：丢包率 = 0%
  - 橙色：丢包率 > 0% 且 < 5%
  - 红色：丢包率 ≥ 5%

#### 2.3.4 统计分析（拨测完成后自动展示）

**Ping 统计指标**

| 分类 | 指标项 |
|------|--------|
| 发包统计 | 总发包数、成功接收数、丢包数、丢包率(%)、最大连续丢包次数 |
| 时延统计 | 最小 RTT、最大 RTT、平均 RTT、P50、P90、P95、P99（单位：ms） |
| 抖动统计 | 平均抖动、最大抖动、RTT 标准差（单位：ms） |

**DNS 统计指标**

| 分类 | 指标项 |
|------|--------|
| 查询统计 | 总查询数、解析成功数、成功率(%)、超时次数、SERVFAIL 次数 |
| 时延统计 | 最小耗时、最大耗时、平均耗时、P50、P90、P99（单位：ms） |
| 解析结果 | IP 变化次数、最后解析 IP、平均 TTL、平均抖动、最大抖动 |

**Curl/HTTP 统计指标**

| 分类 | 指标项 |
|------|--------|
| 请求统计 | 总请求数、成功次数、成功率(%)、超时次数、TLS 错误数、连接错误数、HTTP 2xx/4xx/5xx 分布 |
| 总耗时统计 | 最小值、最大值、平均值、P50、P90、P95、P99（单位：ms） |
| 各阶段平均 | DNS 平均、TCP 平均、TLS 平均、TTFB 平均、平均抖动、最大抖动 |

**TCP 长连接统计指标**

| 分类 | 指标项 |
|------|--------|
| 建连统计 | 建连尝试数、建连成功数、建连失败数、建连成功率(%)、平均/最小/最大建连 RTT |
| 会话稳定性 | 会话总数、断线次数、平均/最长/最短会话时长 |
| 探活 & 重连 | 探活总数、探活成功数、探活失败率(%)、重连次数、平均/最长重连等待时间 |

#### 2.3.5 质量评级

**Ping/DNS/Curl 通用评级**

根据丢包率、平均 RTT、平均抖动三项指标综合评估，取最低评级（木桶效应）：

| 评级 | 丢包率 | 平均 RTT | 平均抖动 |
|------|--------|----------|---------|
| ★★★★★ 优秀 | 0% | < 10ms | < 1ms |
| ★★★★☆ 良好 | < 0.5% | < 30ms | < 3ms |
| ★★★☆☆ 一般 | < 1% | < 50ms | < 10ms |
| ★★☆☆☆ 较差 | < 5% | < 100ms | < 30ms |
| ★☆☆☆☆ 极差 | ≥ 5% | ≥ 100ms | ≥ 30ms |

**TCP 长连接专用评级**

基于建连成功率、平均会话时长、建连 RTT 三个维度综合评估：

| 评级 | 建连成功率 | 平均会话时长 | 建连 RTT |
|------|-----------|------------|---------|
| ★★★★★ 优秀 | 100% | ≥ 5 分钟 | < 10ms |
| ★★★★☆ 良好 | ≥ 95% | ≥ 1 分钟 | < 50ms |
| ★★★☆☆ 一般 | ≥ 90% | ≥ 15 秒 | < 100ms |
| ★★☆☆☆ 较差 | ≥ 50% | ≥ 5 秒 | < 500ms |
| ★☆☆☆☆ 极差 | < 50% | < 5 秒 | ≥ 500ms |

特殊逻辑：
- 从未断线时，会话时长维度自动评为满分
- 智能检测服务端固定超时断连行为（多次断线、建连全成功、会话时长高度一致），给出专项说明

### 2.4 历史记录管理

- 即时拨测和长时间拨测各自独立保留最近 **3 次**拨测记录
- 超过 3 次时自动删除最旧的记录（FIFO 轮换策略）
- 历史记录以按钮形式展示在拨测面板上方，显示序号、时间、协议类型
- 正在进行中的记录标注"(进行中)"
- 点击历史按钮可回看完整拨测数据和统计结果
- 拨测进行中切换查看历史记录不影响当前拨测的数据采集

### 2.5 日志导出

- 支持将完整拨测记录导出为 `.txt` 文本文件
- 导出内容包含：
  - 任务基本信息（类型、协议、目标、时长、时间范围）
  - 全部原始探测记录（格式化表格）
  - 统计分析结果（仅长时间拨测）
  - 质量评级
- 文件名自动建议：`probe_{模式}_{协议}_{目标}_{时间}.txt`

## 3. 非功能需求

### 3.1 用户界面

- 采用现代化扁平化 UI 设计风格（Fusion 主题）
- 主色调：蓝色（#1a73e8）
- 支持高 DPI 屏幕显示
- 强制浅色调色板，确保在 Windows 暗黑模式下也能正常显示
- 字体：优先使用微软雅黑，兼容其他字体回退
- 最小窗口尺寸：900 × 650，默认尺寸：1100 × 780
- Tab 页签式布局切换即时拨测和长时间拨测
- 菜单栏提供帮助和关于对话框

### 3.2 数据存储

- 数据以 JSON + CSV 格式存储在本地文件系统
- 存储目录结构清晰（meta.json + data.csv + stats.json）
- 数据目录位于 exe 同级的 `data/` 文件夹下
- 无需数据库依赖

### 3.3 性能要求

- 拨测引擎在独立线程中运行，不阻塞 UI
- 支持至少 24 小时连续拨测稳定运行
- 拨测间隔精度：±50ms 以内

### 3.4 兼容性

- 目标系统：Windows 10 及以上版本
- Python 版本：3.11+
- 同时兼容 Linux 运行（开发/测试环境）

### 3.5 部署交付

- 最终产物为免安装的 Windows 单文件 exe（约 25 MB）
- 提供多种构建方式：
  - `auto_build.ps1`（PowerShell 全自动构建，推荐）
  - `auto_build.bat`（Bat 全自动构建，双击即可）
  - `build.py`（Python 跨平台构建脚本，支持 `--onefile` 和 `--onedir`）
- 全自动构建：自动下载嵌入式 Python 3.11.9 → 安装依赖 → PyInstaller 打包
- 构建过程一键完成，无需额外配置

## 附录：术语表

| 术语 | 说明 |
|------|------|
| RTT | Round-Trip Time，往返时延 |
| TTL | Time To Live，生存时间 |
| TTFB | Time To First Byte，首字节时间 |
| P50/P90/P95/P99 | 百分位延迟，表示 50%/90%/95%/99% 的请求延迟低于该值 |
| 抖动 (Jitter) | 相邻两次探测 RTT 差值的绝对值 |
| RCODE | DNS 响应码（如 NOERROR、NXDOMAIN、SERVFAIL） |
| ICMP | Internet Control Message Protocol，互联网控制消息协议 |
| TLS | Transport Layer Security，传输层安全协议 |
| TCP Keepalive | TCP 协议层的保活机制，通过发送探测包检测连接是否存活 |
| 指数退避 | Exponential Backoff，重连间隔随失败次数指数增长的退避策略 |
