## 2. 核心技术方案详解

### 2.1 多线程拨测引擎架构

#### 设计思路

拨测引擎是本项目的核心组件。每个引擎（Ping/DNS/Curl）都继承自 `QThread`，在独立线程中执行持续性的网络探测任务，通过 Qt 信号-槽机制将结果异步传递给 UI 层，确保界面流畅不卡顿。

#### 引擎基类模式

三个引擎遵循统一的设计模式：

```python
class XxxEngine(QThread):
    result_ready = pyqtSignal(dict)     # 每条探测结果
    error_occurred = pyqtSignal(str)    # 错误消息
    finished_signal = pyqtSignal()      # 拨测完成

    def run(self):
        self._running = True
        while self._running:
            # 1. 执行探测
            # 2. 解析结果 → dict
            # 3. 发射信号 result_ready.emit(record)
            # 4. 精确等待间隔时间
        self.finished_signal.emit()

    def stop(self):
        self._running = False
```

#### 关键技术点

**1. 精确间隔控制**

使用分段轮询方式实现精确的拨测间隔，同时保证停止响应的及时性：

```python
# 使用短间隔轮询以便及时响应停止
end_wait = time.perf_counter() + sleep_time
while self._running and time.perf_counter() < end_wait:
    time.sleep(0.05)  # 50ms 轮询
```

相比简单的 `time.sleep(interval)`，此方案在调用 `stop()` 后最多 50ms 即可终止线程，而非等待完整间隔。

**2. Ping 引擎的跨平台适配**

Ping 引擎通过检测 `platform.system()` 适配 Windows 和 Linux 两种命令格式：

```python
if system == 'windows':
    cmd = ['ping', '-n', '1', '-w', str(timeout_ms), '-l', str(packet_size), ...]
else:
    cmd = ['ping', '-c', '1', '-W', str(timeout_sec), '-s', str(packet_size), ...]
```

使用正则表达式解析 ping 输出，兼容英文和中文 Windows 的输出格式：

```python
rtt_patterns = [
    r'time[=<](\d+\.?\d*)\s*ms',    # 英文 Windows/Linux
    r'时间[=<](\d+\.?\d*)\s*ms',     # 中文 Windows
    r'rtt.*?=\s*[\d.]+/([\d.]+)',    # Linux summary
]
```

**3. DNS 引擎的容错设计**

DNS 引擎对 dnspython 的多种异常进行了细粒度处理：

| 异常类型 | RCODE | 说明 |
|---------|-------|------|
| `dns.resolver.NXDOMAIN` | NXDOMAIN | 域名不存在 |
| `dns.resolver.NoAnswer` | NOERROR | 无应答记录 |
| `dns.resolver.Timeout` | TIMEOUT | 查询超时 |
| `dns.resolver.NoNameservers` | SERVFAIL | 无可用 DNS 服务器 |

同时对 dnspython 做了导入检查，未安装时给出友好提示而非崩溃。

**4. Curl 引擎的阶段耗时拆分**

HTTP 拨测需要展示各阶段耗时（DNS/TCP/TLS/TTFB/传输），由于 requests 库不直接暴露这些细分数据，采用了混合计时方案：

```python
# 独立计时 DNS 解析阶段
dns_start = time.perf_counter()
socket.getaddrinfo(parsed.hostname, port)
dns_time = (time.perf_counter() - dns_start) * 1000

# 总体请求计时
total_start = time.perf_counter()
response = session.request(...)
total_time = (time.perf_counter() - total_start) * 1000

# 近似拆分各阶段（基于比例估算）
tcp_time = min(elapsed * 0.15, total_time * 0.15)
tls_time = min(elapsed * 0.2, total_time * 0.2) if is_https else 0
ttfb_time = elapsed - dns_time - tcp_time - tls_time
transfer_time = total_time - elapsed
```

### 2.3 数据存储方案

#### 存储策略

采用"文件目录即数据库"的轻量级方案，每次拨测创建独立目录：

| 文件 | 格式 | 用途 |
|------|------|------|
| `meta.json` | JSON | 任务元数据（协议、目标、时间、汇总统计） |
| `data.csv` | CSV | 原始逐条探测记录（追加写入） |
| `stats.json` | JSON | 统计分析结果（仅长时间拨测） |

#### FIFO 轮换机制

每种模式（即时/长时间）最多保留 3 条历史记录，超出时自动删除最旧记录：

```python
def create_record(self, mode, protocol, target, ...):
    existing = self.get_records(mode)
    while len(existing) >= self.MAX_HISTORY:
        oldest = existing.pop(0)
        shutil.rmtree(oldest['_dir'])  # 删除最旧记录
    # 创建新记录...
```

#### CSV 追加写入

使用 `csv.DictWriter` 的追加模式，确保实时写入每条探测记录，即使程序意外退出也不会丢失已采集的数据：

```python
with open(data_path, 'a', newline='', encoding='utf-8') as f:
    writer = csv.DictWriter(f, fieldnames=fieldnames, extrasaction='ignore')
    if not file_exists:
        writer.writeheader()
    writer.writerow(record)
```

### 2.4 统计分析算法

#### 百分位数计算

采用线性插值法计算 P50/P90/P95/P99：

```python
def percentile(sorted_data, p):
    n = len(sorted_data)
    k = (n - 1) * (p / 100.0)
    f, c = math.floor(k), math.ceil(k)
    if f == c:
        return sorted_data[int(k)]
    return sorted_data[int(f)] * (c - k) + sorted_data[int(c)] * (k - f)
```

#### 抖动计算

抖动定义为相邻两次探测 RTT 差值的绝对值：

```python
diffs = [abs(rtt_list[i] - rtt_list[i-1]) for i in range(1, len(rtt_list))]
avg_jitter = sum(diffs) / len(diffs)
max_jitter = max(diffs)
```

#### 最大连续丢包检测

```python
def max_consecutive_loss(statuses):
    max_burst = current_burst = 0
    for s in statuses:
        if s != 'success':
            current_burst += 1
            max_burst = max(max_burst, current_burst)
        else:
            current_burst = 0
    return max_burst
```

#### 质量评级算法

三维度独立评分后取最低分作为综合评级，遵循木桶效应：

```
综合评级 = min(丢包率评分, RTT评分, 抖动评分)
```

## 3. Windows 打包与分发方案

### 3.1 打包方案对比

| 方案 | 产物 | 体积 | 启动速度 | 分发便利性 |

### 3.2 PyInstaller 打包配置要点

#### 隐式导入处理

PyInstaller 静态分析无法识别某些动态导入，需手动声明 hidden imports：

```python
hiddenimports = [
    # dnspython 的各子模块
    'dns.resolver', 'dns.rdatatype', 'dns.name', 'dns.rdata',
    'dns.rdataclass', 'dns.message', 'dns.query', 'dns.exception',
    'dns.zone', 'dns.rcode', 'dns.opcode',
    # requests 相关
    'requests', 'requests.adapters', 'requests.auth',
    'requests.cookies', 'requests.models', 'requests.sessions',
    'urllib3', 'charset_normalizer', 'certifi', 'idna',
]
```

#### 排除不需要的模块

排除常见的大体积科学计算库以减小打包体积：

```python
excludes = ['matplotlib', 'numpy', 'scipy', 'pandas', 'PIL', 'tkinter', 'test', 'unittest']
```

#### PyInstaller frozen 检测

打包后的 exe 运行环境与开发环境不同，需要在入口文件中做特殊处理：

```python
if getattr(sys, 'frozen', False):
    # PyInstaller 打包后运行
    BASE_DIR = os.path.dirname(sys.executable)
    sys.path.insert(0, BASE_DIR)
else:
    # 源码直接运行
    BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
    sys.path.insert(0, BASE_DIR)
```

### 3.3 构建脚本体系

提供了三级构建方案，适应不同场景：

**1. `build.py`（推荐）**  
Python 跨平台构建脚本，自动安装依赖并调用 PyInstaller。支持 `--onefile` 和 `--onedir` 参数。

**2. `build_windows.bat`**  
Windows 批处理一键构建，双击即可运行，适合不熟悉命令行的用户。

**3. `build_portable.bat`**  
高级构建脚本，额外支持下载 Python 嵌入式包构建完全独立的便携版（目标机器无需任何 Python 环境）。

### 3.4 交叉编译限制

**重要说明：** PyInstaller **不支持交叉编译**，即 Linux 环境下无法生成 Windows exe。当前开发环境为 Linux，因此：

- 在 Linux 上已成功验证打包配置的正确性（生成 Linux ELF 可执行文件）
- 最终 Windows exe 需要在 Windows 环境下执行构建脚本生成
- 构建过程已完全自动化，用户只需双击 bat 或运行一行命令

## 4. 代码质量与设计模式

### 4.1 设计模式应用

| 模式 | 应用场景 |
|------|---------|
| 观察者模式 | Qt 信号-槽机制，引擎通过信号通知 UI 更新 |
| 策略模式 | 三种拨测引擎实现相同接口，UI 层按选择实例化 |
| MVC 分层 | UI（View）/ Engine（Controller）/ Storage（Model）清晰分层 |
| 模板方法 | 三个引擎遵循相同的 run/stop 执行框架 |
| FIFO 缓冲 | 历史记录轮换策略 |

### 4.2 错误处理策略

- **引擎层**：所有异常在引擎内部捕获，转换为信号发射，不会导致线程崩溃
- **UI 层**：错误消息以红色文本追加到结果窗口，同时更新状态栏
- **存储层**：文件操作使用 `os.makedirs(exist_ok=True)` 确保目录存在
- **依赖缺失**：DNS 和 HTTP 库使用 `try/except ImportError` 做软依赖检查

### 4.3 线程安全

- 拨测引擎在独立 `QThread` 中运行，通过 Qt 信号队列进行线程间通信
- `_running` 标志位用于优雅停止线程（无需强制终止）
- 窗口关闭时调用 `engine.stop()` + `engine.wait(2000)` 确保线程正确终止
- CSV 文件追加写入无需加锁（单线程写入）

### 5.4 关键决策与权衡

| 决策点 | 选择 | 理由 |
|--------|------|------|
| GUI 框架选择 PyQt6 vs tkinter | PyQt6 | 更现代的外观、更丰富的组件、更好的高 DPI 支持 |
| Ping 实现方式：原生 socket vs 系统命令 | 系统命令 | 原生 ICMP 需要管理员权限，系统命令无此限制 |
| 数据存储：SQLite vs 文件 | 文件（JSON+CSV） | 单机工具无需数据库，文件更轻量、便于查看 |
| 打包方式：单文件 vs 单目录 | 两种都支持 | 单文件便于分发，单目录启动快，用户自选 |
| 历史记录数量限制 | 最多 3 条 | 平衡存储占用与实用性，防止数据无限增长 |

### 5.5 已知限制

1. **Ping 引擎需要系统权限**：某些 Windows 防火墙策略可能阻止 ICMP
2. **Curl 各阶段耗时为近似值**：requests 库不直接暴露 TCP/TLS 分段耗时，采用比例估算
3. **不支持 Linux 下交叉编译 Windows exe**：PyInstaller 的技术限制
4. **长时间拨测不支持 Curl 协议**：考虑到 HTTP 请求资源消耗较大，长时间模式仅支持 Ping 和 DNS
5. **无图表可视化**：当前版本仅提供文本和数值统计，未集成趋势图

## 7. 在 Windows 上构建 exe 的操作指南

### 方法一：Python 构建脚本（推荐）

```bash
# 前置条件：已安装 Python 3.10+
cd Myprogram\netprobe

# 构建单文件 exe（推荐分发）
python build.py --onefile

# 或构建单目录版（启动更快）
python build.py --onedir
```

### 方法二：双击 bat 脚本

1. 双击 `build_windows.bat`
2. 等待 3-5 分钟
3. 在 `dist/` 目录找到生成的 exe

### 方法三：高级便携版

1. 双击 `build_portable.bat`
2. 选择构建方式（PyInstaller 或嵌入式 Python）
3. 按提示完成

### 构建产物说明

- 单文件版：`dist/网络拨测工具.exe`（约 60-80MB），双击即可运行
- 单目录版：`dist/网络拨测工具/`，其中 `网络拨测工具.exe` 为启动入口
- 无需安装 Python 或任何依赖，可直接分发给其他用户
